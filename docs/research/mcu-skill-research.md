## 6개 스킬 후보 분석

### 1. 코드 리뷰/분석 (MCU Code Review)
- **입력**: C 소스 파일 (.c/.h)
- **출력**: 리뷰 리포트 (마크다운 or docx)
- **핵심 체크리스트**:
  - volatile 누락 (ISR 공유 변수)
  - ISR 내 과도한 처리
  - 스택 오버플로우 위험 (큰 로컬 배열)
  - NULL 체크 누락
  - 비원자적 접근 (race condition)
  - 매직 넘버
  - MISRA-C 주요 규칙 위반
- **환경 차이**: 
  - claude.ai: 순수 컨텍스트 기반 분석 (코드를 읽고 판단)
  - Claude Code: cppcheck/clang-tidy 실행 + 컨텍스트 분석 조합
- **실현 가능성**: ★★★★★ (가장 쉬움, 도구 없이도 가능)
- **가치**: ★★★★★ (매일 쓸 수 있음)

### 2. 레지스터 맵/드라이버 코드 생성 (HAL Driver Generator)
- **입력**: 데이터시트 PDF 또는 레지스터 정의 (CSV/텍스트)
- **출력**: HAL 드라이버 코드 (.c/.h)
- **핵심 기능**:
  - 레지스터 맵 구조체 생성 (비트필드, 오프셋)
  - Init/DeInit/Enable/Disable 패턴
  - Read/Write 헬퍼 함수
  - 인터럽트 핸들러 스켈레톤
- **환경 차이**: 거의 없음 (순수 코드 생성)
- **도전 과제**: 
  - 데이터시트 PDF 파싱 정확도
  - MCU 벤더별 레지스터 구조 차이
  - NXP SDK와의 호환성 고려
- **실현 가능성**: ★★★☆☆ (PDF 파싱이 관건)
- **가치**: ★★★★☆ (새 주변장치 작업 시 큰 시간 절약)

### 3. 링커 맵 메모리 분석 (Linker Map Analyzer)
- **입력**: .map 파일 (ARM GCC 링커 출력)
- **출력**: 메모리 사용량 리포트 (표/차트/docx)
- **핵심 기능**:
  - Flash/RAM 사용량 요약
  - 모듈별 메모리 점유 비율
  - 가장 큰 함수/변수 Top N
  - 메모리 한계 대비 경고
  - 이전 빌드 대비 변화량 (선택)
- **환경 차이**:
  - claude.ai: 업로드된 .map 파일 파싱
  - Claude Code: 빌드 후 자동 분석 가능 (hook 연동)
- **파싱 전략**: Python 스크립트로 .map 파싱 → JSON → 리포트
- **실현 가능성**: ★★★★★ (텍스트 파싱, 잘 정의된 포맷)
- **가치**: ★★★★☆ (메모리 제약 MCU에서 필수)

### 4. 테스트 케이스 자동 생성 (Firmware Test Generator)
- **입력**: C 소스 파일 (모듈 헤더/구현)
- **출력**: 테스트 코드 (.c), 테스트 리스트 (xlsx/md)
- **핵심 기능**:
  - 함수 시그니처 분석 → 경계값/정상/에러 케이스
  - Unity/CMock 기반 단위 테스트 생성
  - 테스트 매트릭스 (정상/비정상/경계) 자동 도출
  - 하드웨어 의존성 Mock 생성
- **환경 차이**: 거의 없음
- **도전 과제**:
  - 하드웨어 의존 코드 → Mock 전략
  - 의미 있는 테스트 경계값 도출
  - 기존 테스트 프레임워크(Unity) 호환
- **실현 가능성**: ★★★★☆
- **가치**: ★★★★★ (팀 확장 시 품질 보장에 핵심)

### 5. 코드 기반 기술 문서 생성 (Technical Doc Generator)
- **입력**: C 소스 코드, 설계 문서
- **출력**: 기술 문서 (docx/pdf/md)
- **핵심 기능**:
  - 모듈 구조 분석 → 아키텍처 다이어그램
  - 함수 호출 그래프
  - 타이밍/시퀀스 설명
  - API 레퍼런스
  - Mermaid 다이어그램 자동 생성
- **환경 차이**:
  - claude.ai: docx/pdf 스킬과 연계 용이
  - Claude Code: 소스 트리 전체 스캔 가능
- **기존 자산**: manual-writer 스킬, iTask 문서 경험 활용 가능
- **실현 가능성**: ★★★★★ (기존 스킬/경험 활용)
- **가치**: ★★★★☆ (팀 내 지식 공유에 중요)

### 6. 회로/ADC 정밀도 분석 (Circuit/ADC Precision Analyzer)
- **입력**: 회로도 파라미터, ADC 스펙
- **출력**: 정밀도 분석 리포트 (xlsx/docx)
- **핵심 기능**:
  - ADC 비트 수 대비 이론적 분해능
  - 분압 회로 오차 전파 분석
  - 온도 계수 영향 계산
  - DNL/INL 영향 추정
  - 절연저항 측정 정밀도 (IEEE 43 기준)
- **환경 차이**: 거의 없음 (계산 중심)
- **도전 과제**:
  - 회로도 입력 형식 정의 (파라미터 표? 이미지?)
  - 도메인 지식이 많이 필요
- **실현 가능성**: ★★★☆☆ (입력 형식 설계가 핵심)
- **가치**: ★★★☆☆ (특정 상황에서 매우 유용)

## 우선순위 제안

| 순위 | 스킬 | 이유 |
|------|------|------|
| 1 | 코드 리뷰/분석 | 즉시 활용, 구현 쉬움, 매일 사용 |
| 2 | 링커 맵 분석 | 텍스트 파싱으로 확실한 결과, 자동화 효과 큼 |
| 3 | 테스트 케이스 생성 | 팀 확장(50명) 대비 품질 인프라 |
| 4 | 기술 문서 생성 | 기존 스킬 확장, 지식 공유 |
| 5 | 드라이버 생성 | PDF 파싱 난이도 높지만 가치 큼 |
| 6 | 회로/ADC 분석 | 니치하지만 Accura 2750IRM 등에 특화 |

## 공통 설계 원칙

1. **양쪽 환경 호환**: SKILL.md는 동일, scripts/는 환경에 맞게 분기
2. **점진적 정교화**: 처음엔 간단하게, 피드백으로 개선
3. **입력 유연성**: 파일 업로드 + 텍스트 직접 입력 모두 지원
4. **출력 다양성**: md (빠른 확인) / docx (공식 문서) / xlsx (데이터) 선택
5. **도메인 지식 외부화**: references/ 폴더에 체크리스트, 규칙 분리
